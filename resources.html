Resources & Glossary

A quick-reference glossary for terms used across the course, plus pointers on where to go deeper.

TERM

Agent loop

The Perceive → Reason → Act → Reflect cycle underlying every agent design in this course (Module 1).

TERM

Tool / function calling

A structured request from a model to run a specific, developer-defined function, executed by application code rather than the model itself (Module 4).

TERM

RAG

Retrieval-Augmented Generation — searching an external store for relevant passages and injecting them into the prompt, rather than relying on the model's training data (Module 7).

TERM

ReAct

A prompting pattern that alternates explicit reasoning steps with tool-based actions, observing results in between (Module 3).

TERM

Plan-and-execute

An agent pattern where an explicit multi-step plan is produced before execution begins, optionally revised mid-task (Module 6).

TERM

Orchestrator-worker

A multi-agent pattern where one agent delegates sub-tasks to specialized worker agents and synthesizes their results (Module 9).

TERM

Prompt injection

A security risk where instructions embedded in untrusted content are mistaken by the model for legitimate commands (Module 11).

TERM

Tiered autonomy

Classifying agent actions by risk level — autonomous, confirm-first, or always-human — to scope what runs unsupervised (Module 11).

TERM

Eval set

A representative, versioned set of test cases used to score agent trajectory and outcome quality over time (Module 10).

TERM

Trajectory evaluation

Checking whether an agent took a reasonable path to its answer, not just whether the final answer was correct (Module 10).
