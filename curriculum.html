MODULE 01

What Is an AI Agent, Really?

Before you build anything, you need a definition that survives contact with a whiteboard argument. This module gives you one, and the vocabulary the rest of the course depends on.

PERCEIVE PHASE MODULE 02

The LLM as the Reasoning Core

Every agent is only as good as the decisions its underlying model makes. Understanding what the model is actually doing when it 'reasons' will save you from building on top of a misunderstanding.

REASON PHASE MODULE 03

Prompting Patterns for Agents

System prompts for agents do more work than chatbot prompts ever did -- they define identity, boundaries, available tools, and how to handle uncertainty. Get this wrong and everything downstream inherits the problem.

REASON PHASE MODULE 04

Tool Use & Function Calling

Tools are how an agent stops talking and starts doing. This is the single most important mechanical skill in the whole course -- everything else assumes you can wire this up correctly.

ACT PHASE MODULE 05

Memory: Short-Term, Long-Term & Vector Stores

An agent without memory relearns everything every single turn. Getting memory right is what separates a one-off script from an assistant that actually gets better the more it's used.

PERCEIVE PHASE MODULE 06

Planning: Task Decomposition & Multi-Step Reasoning

Simple tool-calling loops work for one-shot tasks. Real work -- 'plan a launch,' 'refactor this module' -- needs the agent to break a goal into an ordered set of sub-tasks first.

REASON PHASE MODULE 07

Retrieval-Augmented Generation (RAG) for Agents

Agents that reason over your company's actual documents -- not just the model's training data -- need RAG. It's the single most common way to ground an agent in real, current, private information.

PERCEIVE PHASE MODULE 08

A Tour of Agent Frameworks

You now understand the loop well enough to build it by hand -- which means you're also ready to judge which framework (if any) is worth adopting, instead of cargo-culting whatever's trending.

ACT PHASE MODULE 09

Multi-Agent Systems & Orchestration

Some tasks are genuinely too broad for one agent to hold in its head at once. Multi-agent design lets you trade a bigger, murkier problem for several smaller, clearer ones -- at the cost of real coordination overhead.

ACT PHASE MODULE 10

Evaluation, Testing & Debugging Agents

Agents fail in ways traditional software doesn't -- silently, plausibly, and inconsistently across runs. Without a real evaluation practice, you're shipping on vibes.

REFLECT PHASE MODULE 11

Safety, Guardrails & Human-in-the-Loop

The gap between a clever demo and a production agent is almost entirely made of guardrails. This module is the difference between an agent you can trust with real consequences and one you can't.

REFLECT PHASE MODULE 12

Deploying Agents to Production

A working agent in a notebook and a working agent in production are two different engineering problems. This module covers the gap: cost, latency, monitoring, and graceful degradation.

ACT PHASE MODULE 13

The Business Case: ROI, Scoping & Change Management

Technical skill alone doesn't ship an agent. Someone has to decide it's worth building, scope it correctly, and manage what changes for the people whose work it touches.

REFLECT PHASE CAPSTONE

Build a Research & Report Agent

Assemble tool use, memory, RAG, planning, and safety into one working agent.
