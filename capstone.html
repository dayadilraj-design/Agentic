CAPSTONE PROJECT

Build a Research & Report Agent

Given a topic, this agent researches it across your own documents and the web, keeps notes in memory across sessions, produces a cited report, and asks for human confirmation before anything leaves draft form.

BRIEF

What you're building

A single agent that: accepts a research topic, plans its approach, searches both an internal document store (RAG) and, optionally, the web, keeps a running memory of what it has already found, produces a structured report with citations, and never sends or publishes anything without explicit confirmation.

Milestones

M1

Core loop & tool definitions

Stand up the base Perceive-Reason-Act-Reflect loop from Module 1, with a search_docs and a save_note tool defined per Module 4's schema conventions.

M2

RAG over your own documents

Chunk, embed, and index a small document set (Module 7), and confirm the agent can answer a question grounded only in retrieved passages.

M3

Cross-session memory

Add a vector-store-backed memory (Module 5) so the agent recalls prior research on a topic across separate sessions, not just within one conversation.

M4

Plan-and-execute reasoning

Have the agent produce an explicit research plan before executing (Module 6), and log that plan for review.

M5

Safety gate

Apply tiered autonomy (Module 11): searching and note-taking are autonomous; producing and sharing a final report requires human confirmation.

M6

Eval set

Write at least five eval cases (Module 10) covering a straightforward research task, an ambiguous topic, and a topic with no good sources available — and confirm the agent degrades honestly on the last one.

Definition of done

The agent produces a cited, human-reviewed report on a new topic it has never seen, recalls relevant prior research if you ask about a topic a second time, and clearly explains itself — rather than guessing — when it can't find good sources.
